## 2026-03-19 - [XSS via Unsanitized URL in href Attributes]
**Vulnerability:** The `AdminDashboard` component rendered expense evidence URLs directly in `<a>` tag `href` attributes (`href={expense.evidenciaUrl}`). If an attacker could inject a payload like `javascript:alert(1)` into the database (e.g., via a malicious expense submission or API manipulation), clicking the link would execute arbitrary JavaScript in the victim's browser context (Admin session), leading to stored Cross-Site Scripting (XSS).
**Learning:** Any user-provided string that is rendered into a URL context (especially `href` or `src` attributes) is an XSS vector if it is not restricted to safe protocols. React's built-in escaping only protects against HTML injection in text nodes, not protocol-based XSS in attributes.
**Prevention:** Always sanitize URLs before rendering them. Implement and use a safe protocol enforcer (e.g., `getSafeUrl(url)` that verifies the protocol is `http:` or `https:`) for all user-supplied links.
